What Is This Project?

Stitch Smart is a luxury apparel e-commerce web application — a complete online clothing store built as a Final Year Project (FYP) at a Pakistani university.

Customers can:

  • › Browse and buy ready-made garments (hoodies, sweatpants, crewnecks, shorts)
  • › Customize their own garment using a step-by-step Design Yourself form
  • › Ask an AI-powered chatbot about products, sizing, and availability
  • › Create accounts, manage orders, write product reviews, manage wishlists
  • › Pay via JazzCash (Pakistan's top mobile wallet) or Cash on Delivery

Administrators can:

  • › Manage all products, categories, banners, and CMS pages
  • › View orders and update their status
  • › Access a dashboard with revenue analytics
  • › Use an AI assistant (Google Gemini) to generate product descriptions
  • › Configure website theme dynamically from the admin panel

Why PHP? Why Not Other Languages?

Why PHP is the right choice

  • XAMPP bundles Apache + MySQL + PHP — zero extra setup for the team
  • Runs natively inside Apache via mod_php (no extra port or process manager)
  • Built-in superglobals make MVC trivial without a heavy framework
  • mysqli extension is built-in — no ORM or driver installation needed
  • Shared PHP hosting is the cheapest globally — cPanel hosts run it instantly
  • Team already had PHP knowledge — no learning curve mid-project

Why others were rejected

  • Node.js — npm complexity, PM2 needed, async hell for traditional server-rendered HTML
  • Python/Django — Python already used for AI service; running two Python servers wastes memory
  • Laravel — 30MB+ dependencies, Eloquent ORM, Artisan CLI steep learning curve for FYP scope
  • Java/Spring — verbose annotations, DI config, Maven — doubles development time with no benefit
  • Ruby/Rails — no team knowledge, not common in Pakistan's hosting ecosystem
Conclusion: PHP 8.0+ is modern, fast, natively supported on XAMPP, cheap to host, and perfectly suited for a university-level e-commerce project demonstrable from a laptop.

Why Google Gemini API? Why Not Other AIs?

Two AI integrations exist. Admin AI — PHP calls Gemini REST via cURL to generate product descriptions. Chatbot AI — Python calls Gemini via LangChain for customer Q&A responses.

Google Gemini 1.5 Flash ✓
Chosen
  • 1,500 free requests/day — no credit card needed
  • Optimized for low latency (2–3 second responses)
  • Accessible from Pakistan without VPN
  • Official langchain-google-genai package
  • High-quality natural language for product copy
OpenAI GPT-4 ✗
Rejected
  • Requires paid API key (credit card)
  • No free tier for GPT-3.5 Turbo anymore
  • Intermittent access issues from Pakistan
  • GPT-4 costs ~$0.03/1K tokens → $5–15/day at FYP scale
Anthropic Claude ✗
Rejected
  • Very limited free tier
  • Less mature LangChain integration at time of development
  • International payment setup difficult for students
Local LLM (Llama/Mistral) ✗
Rejected
  • Requires CUDA GPU — team machines are CPU only
  • CPU inference is 30–60 seconds per response
  • Hugging Face cloud API free tier is very restrictive
Embeddings note: sentence-transformers (all-MiniLM-L6-v2) runs 100% locally for generating product vectors — no API cost, no latency. This is why OpenAI Embeddings or Gemini Embeddings were not used for the FAISS index.

Why MySQL? Why Not Other Databases?

Why MySQL / MariaDB

  • XAMPP bundles MariaDB (MySQL-compatible) — zero setup
  • PHP's mysqli extension is built-in, no driver install
  • Data is relational: orders → items → products → categories
  • phpMyAdmin provides visual DB management (also in XAMPP)
  • Most common PHP database — abundant documentation

Alternatives rejected

  • PostgreSQL — not bundled with XAMPP, requires separate install
  • MongoDB — our data is relational; NoSQL adds manual join complexity
  • SQLite — single-file, unreliable under concurrent web writes
  • Firebase — requires Google billing, real-time overhead not needed here

How the Application Works

PHP MVC Request Flow:

Browser ──► Apache ──► public/index.php (Router) │ reads ?page=products │ matches ──► ProductController::index() │ calls Product model (SQL) │ MySQL returns rows │ controller passes $data to views/products.php │ view renders HTML ──► Browser

AI Chatbot RAG Pipeline:

Customer types: "do you have hoodies under Rs 3000?" │ ▼ chatbot.js ──► AJAX POST ──► ChatController.php │ HTTP POST ──► FastAPI :5000/chat │ rag_chain.py ╱ ╲ vector_store.py Google Gemini API FAISS top-5 similar generates response products from index using product context ╲ ╱ full_response text │ PHP saves to chat_history DB table │ JSON returned to chatbot.js ──► displayed to customer

Admin AI Description Generation:

Admin clicks "Generate Description" button │ ▼ ProductController.php ──► cURL POST ──► Gemini REST API │ ▼ Gemini returns product description text │ ▼ PHP echoes it into the textarea — admin can edit and save

What is RAG? Instead of asking AI to answer from its generic training data, we build a searchable index of YOUR products. When a customer asks a question, we find relevant products first, then give the AI question + product context → the AI answers using real store data.

StepWhat HappensFile Responsible
IndexingAdmin exports products.json → Python builds FAISS index using sentence-transformersvector_store.py
Question receivedCustomer types question → chatbot.js AJAX → ChatController.php → FastAPIroutes.py
RetrievalQuestion embedded → FAISS returns top-5 similar productsrag_chain.py
GenerationPrompt = system + product context + history + question → Gemini LLMrag_chain.py + prompts.py
ResponseAnswer returned → saved to DB chat_history → displayed in widgetChatController.php
Greeting shortcut: Inputs like "hi", "hello", "hey" skip FAISS retrieval — the AI responds conversationally without searching the product index, reducing unnecessary API calls.

Payment System — JazzCash

JazzCash is Pakistan's most popular mobile wallet (40+ million users, owned by Jazz Telecom).

StepWhat Happens
1. RegisterCustomer enters Jazz mobile number + sets 4–6 digit MPIN inside Stitch Smart
2. StoreMPIN is hashed with password_hash() (bcrypt) and stored in jazzcash_accounts table
3. CheckoutCustomer selects "JazzCash" as payment method
4. VerifyCustomer enters mobile + MPIN → system runs password_verify() against stored hash
5. ProcessIf verified: order placed, transaction recorded in jazzcash_transactions, balance debited

Why JazzCash

  • 40+ million Pakistani users — largest local mobile wallet
  • No international payment method required
  • Real Pakistani e-commerce payment ecosystem
  • Demonstrates full payment UX for FYP purposes

Why not Stripe / PayPal

  • Stripe has no Pakistani merchant account support
  • PayPal Pakistan is receive-only (cannot process payments)
  • Both require USD international bank accounts
  • Neither is used by typical Pakistani online shoppers
Note: This is a simulated JazzCash integration. A real merchant integration requires a JazzCash Business Account + official REST API access (requires NTN/CNIC business verification). The current implementation demonstrates the complete UX flow and data model correctly — appropriate for an FYP.

Glossary

MVC
Model-View-Controller — separates database logic (Model), HTML templates (View), and business logic (Controller)
RAG
Retrieval Augmented Generation — AI searches a local knowledge base first, then generates an answer using that specific data
FAISS
Facebook AI Similarity Search — library for lightning-fast vector nearest-neighbor search (finds "similar" items)
Embedding
Converting text into a list of numbers (a vector) that encodes meaning, allowing mathematical similarity comparison
LLM
Large Language Model — AI model trained on vast text that generates human-like responses (e.g. Gemini, GPT)
FastAPI
Python web framework for building APIs. Used for the chatbot microservice running on port 5000
CSRF Token
Cross-Site Request Forgery token — hidden secret in forms that proves the form came from our own website
Bcrypt
Strong one-way password hashing algorithm — makes stored passwords uncrackable even if the database is stolen
XAMPP
Free software bundle (Apache + MySQL + PHP) for running web applications locally on Windows/Mac
PHPMailer
PHP library that handles sending emails via SMTP, including TLS authentication and MIME encoding
SMTP
Simple Mail Transfer Protocol — the internet standard for sending email between servers
JazzCash
Pakistan's most popular mobile wallet, owned by Jazz Telecom, with 40+ million registered users
MPIN
Mobile PIN — a 4–6 digit code customers use to authenticate and authorize JazzCash transactions
LangChain
Python framework for building LLM applications — handles prompt templates, retrieval chains, and memory
sentence-transformers
Python library converting sentences to vectors using pre-trained transformer models — runs locally for free
CMS
Content Management System — ability to edit website pages and content from an admin panel without writing code
FYP
Final Year Project — the capstone academic project completed in the last year of an undergraduate degree
CRON
Scheduled tasks that run automatically at set intervals (e.g. rebuild product index every night at 2am)
